# Archive

This directory contains historical, obsolete, or legacy configurations, scripts, and documentation that have been relocated to avoid conflicts with the unified, official repository architecture.

## Contents
- **`configs/`**: Legacy YAML configurations used during initial exploratory sweeps and description function (DF) tests under earlier code versions.

## Status
All active and official methodology runs, protocols, and validation checks must use the unified Caputo order and parameters defined under `version_2/configs/unified_caputo_protocol.json`. Do not run legacy configurations in `archive/` as official routes.
