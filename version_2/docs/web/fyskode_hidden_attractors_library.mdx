Esta pagina presenta la primera version documental de `hidden-attractors-fo`,
una libreria Python organizada para reproducir, auditar y extender experimentos
numericos con candidatos a atractores ocultos en sistemas de Chua fraccionarios.

La idea no es esconder el origen de los metodos dentro del codigo. Cada bloque
de calculo queda asociado con su referencia: Leonov--Kuznetsov para la
distincion entre atractores ocultos y autoexcitados, Danca para el ejemplo de
Chua fraccionario no suave, Matignon para estabilidad local fraccionaria,
Benettin para exponentes de Lyapunov, y Caputo para la derivada fraccionaria
causal.

## Que hace la libreria

La libreria separa cuatro tareas:

- definir el sistema de Chua no suave;
- cargar trayectorias reales generadas por los workflows;
- graficar espacio de fases, proyecciones, series temporales y diagramas de bifurcacion;
- documentar que diagnosticos son evidencia numerica y cuales no prueban ocultedad.

El contrato de trayectoria usado por los ejemplos es simple:

```text
t,x,y,z
```

Con eso se puede cargar una trayectoria CSV del proyecto y generar figuras:

```python
from hidden_attractors.io import load_trajectory_csv
from hidden_attractors.plotting import plot_phase_space

trajectory = load_trajectory_csv("outputs/.../reference_attractor.csv")
plot_phase_space(trajectory, "outputs/my_case/phase_space_3d.png")
```

## Lectura cientifica conservadora

Una trayectoria bonita no basta para declarar un atractor oculto. La secuencia
documentada es:

```text
semilla armonica
-> atractor observado
-> comparacion con target
-> robustez numerica
-> pruebas desde vecindades de equilibrios
-> lectura de ocultedad
```

Si aparece una trayectoria `TARGET` desde una vecindad de equilibrio, el
candidato no debe llamarse oculto bajo ese contrato. Esa separacion es la que
hace que la libreria sea util como herramienta cientifica y no solo como
generador de figuras.
