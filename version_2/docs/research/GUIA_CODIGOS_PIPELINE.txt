CURRENT UNIFIED PROTOCOL PIPELINE GUIDE / GUIA ACTUAL DEL PIPELINE DE PROTOCOLO UNIFICADO
Hidden Attractors Fractional Order - version_2

Last updated: Complete migration to the canonical unified Caputo protocol. / Última actualización: Migración completa al protocolo canónico unificado de Caputo.
All search, continuation, and validation of hidden attractors follows a strict order of 10 stages. / Toda la búsqueda, continuación y validación de atractores ocultos sigue un orden estricto de 10 etapas.

1. MAIN RULE / REGLA PRINCIPAL
==============================

The library is used from `hidden_attractors/`. Obsolete or "legacy" scripts must not be used or extended. / La librería se usa desde `hidden_attractors/`. Los scripts obsoletos o "legacy" no deben usarse ni ampliarse.
Operational calculation rule: / Regla operativa de cálculo:
    - Lightweight preparation, algebraic seeds, state reading, and metrics reporting: Python API. / Preparación ligera, semillas algebraicas, lectura de estados y reporte de métricas: Python API.
    - Large sweep integration, basins, hiddenness tests, and Lyapunov exponents: C backends (FractionalChuaBackend, BasinBackend). / Integración de sweeps grandes, cuencas, pruebas de ocultedad y Lyapunov: C backends (FractionalChuaBackend, BasinBackend).
    - No seed is rejected for being periodic prior to continuation (pre_continuation_periodic). / No se rechaza ninguna semilla por ser periódica antes de la continuación (pre_continuation_periodic).

2. CANONICAL UNIFIED COMMAND (hidden-attractors protocol) / COMANDO CANÓNICO UNIFICADO (hidden-attractors protocol)
===================================================================================================================

The only promoted entrypoint for validation is the unified subcommand `hidden-attractors protocol` (or the `hidden-attractors-protocol` CLI wrapper), which exposes 8 mandatory subcommands corresponding to the official stage order:
El único entrypoint promocionado para la validación es `hidden-attractors protocol` (o el wrapper CLI `hidden-attractors-protocol`), el cual expone 8 subcomandos obligatorios correspondientes al orden de etapas oficiales:

    hidden-attractors protocol generate-seeds --help
    hidden-attractors protocol soft-precheck --help
    hidden-attractors protocol continue --help
    hidden-attractors protocol filter-survivors --help
    hidden-attractors protocol build-reference --help
    hidden-attractors protocol robustness --help
    hidden-attractors protocol hiddenness --help
    hidden-attractors protocol diagnostics --help

Each command generates or processes a machine-readable summary.json with the same common metadata structure (schema_version, protocol_version, stage, etc.).
Cada comando genera o procesa un summary.json machine-readable con la misma estructura común de metadatos (schema_version, protocol_version, stage, etc.).

3. COMPLETE PROTOCOL SEQUENCE / SECUENCIA COMPLETA DEL PROTOCOLO
=================================================================

Example of sequential execution using a contract (e.g., configs/unified_caputo_protocol.json):
Ejemplo de ejecución secuencial usando un contrato (ej: configs/unified_caputo_protocol.json):

  a) Seed Search and Generation / Búsqueda y Generación de Semillas:
     hidden-attractors protocol generate-seeds --contract configs/unified_caputo_protocol.json --output outputs/run/03_seed_generation/summary.json

  b) Soft Precheck (Diagnostic, does not reject for periodicity) / Precheck Suave (Diagnóstico, no rechaza por periodicidad):
     hidden-attractors protocol soft-precheck --contract configs/unified_caputo_protocol.json --payload outputs/run/03_seed_generation/summary.json --output outputs/run/04_soft_precheck/summary.json

  c) Homotopy Continuation by Parameter lambda / Continuación por Parámetro de Homotopía lambda:
     hidden-attractors protocol continue --contract configs/unified_caputo_protocol.json --candidate-id candidate_001 --lambda-values 0,0.25,0.5,0.75,1 --output outputs/run/05_continuation/summary.json

  d) Post-Continuation Filtering (Periodic orbits discarded here) / Filtrado Post-Continuación (Aquí se descartan órbitas periódicas):
     hidden-attractors protocol filter-survivors --contract configs/unified_caputo_protocol.json --payload outputs/run/05_continuation/summary.json --output outputs/run/06_post_continuation_filter/summary.json

  e) Dynamic Reference Creation / Creación de Referencia Dinámica:
     hidden-attractors protocol build-reference --contract configs/unified_caputo_protocol.json --payload outputs/run/06_post_continuation_filter/summary.json --output outputs/run/07_dynamic_reference/summary.json

  f) Robustness Tests (Step h changes, Lm, full vs truncated memory) / Pruebas de Robustez (Cambios de paso h, Lm, full vs truncated memory):
     hidden-attractors protocol robustness --contract configs/unified_caputo_protocol.json --payload outputs/run/07_dynamic_reference/summary.json --output outputs/run/08_robustness/summary.json

  g) Hiddenness Tests (Inner ball sampling and basin slices) / Pruebas de Ocultedad (Muestreo interior en bolas y cortes de cuenca):
     hidden-attractors protocol hiddenness --contract configs/unified_caputo_protocol.json --payload outputs/run/08_robustness/summary.json --output outputs/run/09_hiddenness_tests/summary.json

  h) Complementary Diagnostics (FFT, PSD, Lyapunov) / Diagnósticos Complementarios (FFT, PSD, Lyapunov):
     hidden-attractors protocol diagnostics --contract configs/unified_caputo_protocol.json --payload outputs/run/09_hiddenness_tests/summary.json --output outputs/run/10_diagnostics/summary.json

Toda la configuración pasa ahora por `unified_caputo_protocol.json`.
  - Danca ABM replicas remain under `tools/legacy/danca2017_chua_abm_replication.py` only for historical benchmark and comparison validation. / Las réplicas de Danca ABM permanecen bajo `tools/legacy/danca2017_chua_abm_replication.py` únicamente para validación histórica de benchmark y comparación.
  - Pre-continuation periodicity is no longer a cause for rejection, guaranteeing search space preservation for Lur'e transport. / La periodicidad antes de la continuación ya no es causa de descarte, garantizando la preservación del espacio de búsqueda para transporte mediante Lur'e.
