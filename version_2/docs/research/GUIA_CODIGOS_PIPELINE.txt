GUIA ACTUAL DEL PIPELINE
Hidden Attractors Fractional Order - version_2

Ultima actualizacion: migracion a API publica de Python con backends C como
ruta obligatoria para calculo numerico pesado.


1. REGLA PRINCIPAL
==================

La libreria se usa desde `hidden_attractors/`. Los scripts de `tools/legacy/`
no deben crecer como API nueva.

Regla operativa:

    - Preparacion ligera, lectura de resultados, modelos, semillas armonicas,
      metricas, tablas y graficas: Python API.
    - Integracion EFORK, verificacion de ocultedad, cuencas, bifurcaciones y
      Lyapunov operacional: backend C.
    - No se debe activar una ruta pesada en Python cuando existe la ruta C.

La opcion historica `HIDDEN_ATTRACTORS_PYTHON_BASIN` queda retirada. Los cortes
de cuenca xy/xz/yz se calculan con `chua_basin_lib.c`.


2. ENTRADA RECOMENDADA
======================

Desde `version_2`:

    python -m pip install -e .
    python -m compileall hidden_attractors examples tests tools/cli
    python examples/quickstart_equilibria.py
    python examples/list_final_candidates.py

Comando recomendado para el pipeline fraccionario unificado:

    hidden-attractors-unified-chua --help

Equivalente sin instalar scripts de consola:

    python -m hidden_attractors.workflows.unified_chua --help

Este wrapper recibe argumentos normales y llama al pipeline mantenido sin que
el usuario tenga que preparar `$env:...`.


3. COMANDOS CANONICOS SIN VARIABLES DE ENTORNO
==============================================

Corrida balanceada del Chua no suave:

    hidden-attractors-unified-chua --model piecewise --run-mode balanced --q 0.9998

Corrida rapida:

    hidden-attractors-unified-chua --model piecewise --run-mode fast --q 0.9998

Solo cuencas desde el resumen previo:

    hidden-attractors-unified-chua --model piecewise --run-mode basin_only --basin-grid 360x360 --basin-planes

Sweep de q:

    hidden-attractors-unified-chua --model piecewise --run-mode q_sweep --q-values 0.97,0.98,0.985,0.99,0.995,1.0

Comparacion DF clasica contra Machado:

    hidden-attractors-unified-chua --model piecewise --run-mode df_compare --machado-mu 0.5

Sweep Machado rapido:

    hidden-attractors-unified-chua --model piecewise --run-mode machado_sweep_fast --output-dir runs_machado_sweep_fast

Activar PSD de Welch:

    hidden-attractors-unified-chua --model piecewise --run-mode balanced --spectral --psd

Activar Lyapunov operacional C:

    hidden-attractors-unified-chua --model piecewise --run-mode balanced --lyapunov

Exportar series para TISEAN:

    hidden-attractors-unified-chua --model piecewise --run-mode balanced --tisean

Control de workers C:

    hidden-attractors-unified-chua --run-mode balanced --basin-workers 8 --bif-workers 8 --native-efork-workers 8


4. API PUBLICA YA ACTIVADA
==========================

Modelo de Chua y equilibrios:

    from hidden_attractors.models import chua_piecewise_parameters, equilibria_piecewise

Semillas por funcion descriptiva clasica o Machado:

    from hidden_attractors.seed_generation import find_harmonic_seed

    classic = find_harmonic_seed(q=0.9998, method="classic", branch_index=0)
    machado = find_harmonic_seed(q=0.9998, method="machado", mu=2.0)

Semilla sesgada Lur'e:

    from hidden_attractors.seed_generation import reconstruct_biased_lure_seed

    biased = reconstruct_biased_lure_seed(
        q=0.9998,
        amplitude=5.0,
        sigma0=0.0,
        omega=2.04,
    )

Ventana de historia fraccionaria para contratos EFORK:

    from hidden_attractors.solvers import FractionalHistory

    history = FractionalHistory.from_trajectory(
        trajectory,
        q=0.9998,
        h=0.01,
        memory_length=8.0,
    )
    efork_history = history.as_efork_history()

Backends C:

    from hidden_attractors.native import FractionalChuaBackend, BasinBackend

    efork = FractionalChuaBackend.build()
    basin = BasinBackend.build()

6. ESTADO DE `tools/legacy`
===========================

No se puede borrar toda la carpeta `tools/legacy/` todavia.

Motivo:

    - El wrapper `hidden-attractors-unified-chua` sigue delegando al script
      amplio `tools/legacy/unified_nyquist_hidden_pipeline.py`.
    - La replica ABM de Danca sigue en
      `tools/legacy/danca2017_chua_abm_replication.py`.
    - Algunas busquedas historicas y reportes reproducibles siguen leyendo
      artefactos producidos por scripts legacy.

Lo que ya no debe usarse como ruta nueva:

    - Activar variables `$env:HIDDEN_ATTRACTORS_*` manualmente.
    - Usar rutinas pesadas de cuenca en Python.
    - Crear nuevos scripts en `tools/legacy/` cuando la logica pueda ir en
      `hidden_attractors/`.

Lo que si quedo adaptado:

    - DF centrada, Machado y semilla sesgada: `hidden_attractors.seed_generation`.
    - Historia EFORK transportable: `hidden_attractors.solvers.FractionalHistory`.
    - Backends C empaquetados: `hidden_attractors.native`.
    - Entrada de pipeline sin `$env`: `hidden_attractors.workflows.unified_chua`.

Pendiente antes de poder borrar `tools/legacy/`:

    1. Migrar completamente `unified_nyquist_hidden_pipeline.py` a modulos
       internos pequenos bajo `hidden_attractors.workflows`.
    2. Migrar o aislar la replica Danca ABM como workflow publico.
    3. Reemplazar continuaciones multiparametro que todavia llaman EFORK Python
       por llamadas al backend C.
    4. Validar que ningun reporte, notebook o prueba dependa de rutas legacy.


7. REGLA PARA NUEVO CODIGO
==========================

Si el codigo calcula muchas trayectorias, barre una malla, clasifica cuencas,
estima Lyapunov o repite integraciones largas, debe llamar a C.

Si el codigo solo arma parametros, calcula una semilla algebraica, carga CSV,
calcula metricas de una trayectoria ya producida o genera una figura, puede
vivir como Python puro dentro de `hidden_attractors/`.
